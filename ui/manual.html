1. What Axon is

Axon is your local AI operator. That means it lives on your computer, keeps your workspaces in mind, and helps you think, plan, do, and check. It is not just a chat box. It is an operator console.

Axon lives on your machine

Your files, missions, and settings stay local by default. Axon uses your own computer as its main home.

Axon can think and act

It can inspect files, run safe actions, show progress, and tell you what happened next.

Axon can still use optional outside helpers

If you enable cloud providers, Axon can route work there too. Local comes first.

Simple version
Axon is like a smart helper that sits on your computer. Your phone can talk to it, but your computer is the one doing the work.

2. How Axon thinks

Axon always follows the same safety-minded loop. This is the heart of the product.

Observe
Look first
Axon checks your goal, current workspace, runtime state, and what it already knows.
Plan
Think next
Axon decides which steps make sense before it starts touching anything.
Execute
Do the work
Axon uses local tools, models, or approved cloud helpers to carry the plan out.
Verify
Check the result
Axon confirms what changed, what worked, and what still needs help.
Recover
Stay calm if something breaks
Axon reports the problem, falls back safely, and shows the next safe move.

6. Secure mobile access

Axon lets your phone in carefully. It does not just trust every browser automatically.

Simple security story
  • PIN lock protects the app.
  • A session token is created after sign-in.
  • Axon accepts both X-Axon-Token and legacy X-DevBrain-Token headers.
  • Files are sandboxed to your home directory.
  • Vault secrets stay local and only enter approved actions.
  • Risky actions use guided confirmation.
PIN and session flow
1. You open Axon on your phone.
2. You enter your PIN.
3. Axon creates a session token.
4. Your phone sends that token on future requests.
5. Axon allows the request only if the session is still valid.

7. Stable domain and PWA reliability

A PWA is happiest when it keeps the same web address. That is why Axon should eventually live behind a stable HTTPS origin like axon.edudashpro.org.za.

Stable origin plan

Why this matters
  • If the address keeps changing, your PWA feels confused.
  • If the origin stays stable, reconnect logic gets simpler.
  • The user understands whether they are connected, reconnecting, or offline.
  • Service worker caching behaves much more predictably.

9. Local models and cloud models

Think of this as two kinds of brains.

Local brain

Private. Runs on your computer through Ollama. This is Axon's default path.

Code -> qwen2.5-coder General -> qwen3 Reasoning -> deepseek-r1 Embeddings -> nomic-embed-text Vision -> local route when available
Cloud helper

Optional. Outside provider. Only used when you enable it or choose it.

Anthropic OpenAI GPT-style Gemini Gem-style Generic API
Simple rule
Local means private brain on your computer. Cloud means optional outside helper. Axon uses local first.

11. Dash integration

Dash and Axon play different roles. Dash talks to people. Axon gets work done.

  • Dash can hand trusted work to Axon.
  • Axon can run local workspaces, files, scans, and mission actions.
  • Axon can return summaries, files, reminders, and status back to Dash.

12. Important commands

These are real commands and paths Axon supports today. Use the guided Settings UI when you want safer buttons. Use the terminal when you want direct control.

Start, stop, open
axon
Starts Axon using the local launcher. Use this for normal day-to-day startup.
devbrain
Legacy compatibility launcher. Still supported while the storage path remains compatible.
~/.devbrain/start.sh
Starts Axon directly from the install folder.
~/.devbrain/stop.sh
Stops the local Axon server.
http://localhost:7734
Opens the local app in a browser.
http://localhost:7734/manual
Opens this manual in the local Axon server.
Runtime and models
~/.devbrain/ollama-start.sh status
Checks whether Ollama is running and whether it is using systemd or manual mode.
~/.devbrain/ollama-start.sh start
Starts Ollama if you are managing it from the terminal.
ollama list
Shows installed local models.
ollama pull qwen2.5-coder:1.5b
Pulls a local model directly from Ollama. Use this when you want to preload a small coding model fast.
curl http://localhost:11434/api/version
Simple runtime health check for Ollama.
Mobile and tunnel
~/.devbrain/tunnel.sh start
Starts the HTTPS tunnel. Best when you want phone access or PWA install.
~/.devbrain/tunnel.sh status
Checks whether the tunnel is running.
~/.devbrain/tunnel.sh url
Prints the current tunnel URL if one exists.
~/.devbrain/tunnel.sh stop
Stops the tunnel.
Settings -> Mobile
Best beginner-friendly way to get QR codes, LAN info, Tailscale info, and tunnel state.
Developer and maintenance
curl http://localhost:7734/api/health
Checks if the Axon server is alive.
tail -f ~/.devbrain/devbrain.log
Watch the Axon server log in real time.
tail -f ~/.devbrain/ollama.log
Watch the Ollama log in real time.
ls ~/.devbrain
Shows the Axon install folder, logs, database, scripts, and UI files.
Settings -> System Actions
Safest way to restart Axon, prepare an Ollama restart, or review machine reboot commands.
