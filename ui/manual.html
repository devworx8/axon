1. What Axon is

Axon is a local AI operator. It runs a FastAPI server on your computer, keeps a local SQLite state store, scans your workspaces, tracks missions, stores reusable playbooks, and gives you an operator console for local and optional cloud-assisted workflows.

🧠
Operator console

The Console is where you give Axon goals, tasks, and commands. It can observe, plan, execute, verify, and recover.

🗂️
Workspace awareness

Axon tracks workspace health, TODO debt, git age, notes, playbooks, and mission context so it can act with local state already loaded.

🔐
Privacy-first by default

Local models are the default route. External adapters are optional, clearly marked, and disabled until you enable them.

Core services

  • FastAPI server: serves the UI, API routes, streaming responses, and system actions.
  • SQLite state: stores workspaces, missions, playbooks, activity, and settings under the existing ~/.devbrain path for compatibility.
  • Scanner + scheduler: keep workspace health and morning brief state current.
  • Secure Vault: stores secrets locally and only unlocks them into approved actions.

Runtime model routing

  • Default runtime: Ollama.
  • Role-based routes: Code, General, Reasoning, Embeddings, Vision.
  • Fallback behavior: Axon prefers configured local models, then auto-matches installed families.
  • Cloud adapters: optional, external, and disabled by default.
Default posture: local-first, privacy-first, and role-aware. If you never enable a cloud adapter, Axon still works as a local operator.

3. Getting started

Step 1
Start Axon
axon

The legacy devbrain command still works too. Both point at the same compatible runtime path.

Step 2
Set your workspaces root

Go to Settings → Runtime and set the folders Axon should scan, for example ~/Desktop or a custom projects directory.

Step 3
Run Axon Scan

Use the sidebar action to discover workspaces, calculate health, refresh mission suggestions, and populate the Console context.

Local runtime setup

Axon is happiest when Ollama is running locally. In Settings, pick your default Console model, then tune the role routes under Local Models.

If you need cloud help, you can switch runtime paths later without changing your data store.

Compatibility note

Axon still uses the existing ~/.devbrain storage path and database so nothing breaks while the product language evolves.

4. Console

The Console is Axon’s primary operator surface. You can give it a goal, task, or direct command and it will use local context, model routing, and tool access to respond.

Axon Console
Observe Plan Execute Verify Recover
Give Axon a goal, task, or command...

Agent phases

  • Observe: inspect the request, workspace scope, and runtime state.
  • Plan: decide what to inspect or execute next.
  • Execute: use tools or model calls to do the work.
  • Verify: confirm the result, summarize output, or sanity-check the action.
  • Recover: surface errors, fall back, or suggest safe next actions.

Operator roles

  • Planner Agent
  • Coder Agent
  • Scanner Agent
  • Reviewer Agent
  • Repair Agent

5. Workspaces

Workspaces replace the old Projects surface. Axon scans them, scores health, tracks TODO debt, notes git age, and lets you inspect or open a focused Console from each card.

Inspect

Runs a focused workspace analysis and surfaces risks, debt, and recommended next actions.

Open Console

Scopes the Console to one workspace so Axon can reason with tighter context.

Health signals

Readiness bars, TODO count, branch age, and optional GitHub status stay visible on the card.

7. Playbooks

Playbooks replace the old prompt library. Use them for reusable instructions, workspace-specific workflows, deployment steps, review checklists, or external agent handoff templates.

8. Timeline

Timeline replaces the old Activity view. It groups events like scan, agent, system, model, mission, and vault.

Morning Brief

The Morning Brief is Axon’s daily operator summary. It pulls from workspace state, mission urgency, timeline events, and runtime readiness.

9. Secure Vault

Secure Vault preserves the local-only security posture. Axon stores secrets locally and only injects them into approved actions.

Approved actions only: the permission system keeps vault access, system actions, cloud adapters, and local write behavior explicit.

10. Remote mobile use

Axon can be used from your phone through LAN, Tailscale, or an HTTPS tunnel. The phone UI is a mobile web app / PWA surface, while the real work still runs on your computer.

LAN

Best for same-Wi-Fi use. Open the local address shown in the Mobile sheet.

Tailscale

Best for private remote use. Works anywhere your tailnet is reachable.

HTTPS tunnel

Best for browser/PWA install. Use the Mobile modal to start the tunnel and open the secure URL.

11. External specialist agents

Axon is local-first, but it now has a clean place for optional cloud adapters when you need specialist help or cross-platform agent workflows.

Local model roles

  • Code → preferred family qwen2.5-coder
  • General → preferred family qwen3
  • Reasoning → preferred family deepseek-r1
  • Embeddings → preferred family nomic-embed-text
  • Vision → preferred family llava

Optional local alternates include codegemma, starcoder2, and codellama.

Optional external adapters

  • OpenAI GPT-style agents
  • Gemini Gem-style agents
  • Generic API providers

These adapters are disabled by default, clearly marked external, and do not affect local workflows unless you turn them on.

12. Troubleshooting

Axon opens but local models do not respond

Check Settings → Runtime to confirm Ollama is reachable, then verify the saved Console model and role routes under Local Models.

Cloud adapters show disabled

That is expected by design. Enable Cloud Agents first, then toggle the specific external adapter you want to use.

Phone cannot install the PWA

Use the HTTPS tunnel from the Mobile modal. PWA install requires a secure origin, so LAN HTTP alone is not enough for installation.

Display goes blank during large Ollama runs

On display-attached low-VRAM NVIDIA cards, large 7B GPU loads can destabilize the desktop. Axon now adds a safety warning and safer fallback behavior, but smaller local models are still the safest path on those hosts.
