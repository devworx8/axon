1. What is Axon?

Axon is a local AI Operator that lives on your computer — calm, private, and local-first. Think of it as an operating console for your work: it watches your workspaces, tracks missions, stores reusable playbooks, and briefs you every morning on what matters most.

😩
The problem

Workspaces going stale, missions forgotten in notes apps, playbooks lost in chat history, no clear operator view of what to do today.

→
🧠
The solution

One local operator console that knows your workspaces, tracks missions by priority, stores playbooks, and tells you every morning exactly what to do.

What Axon can do for you

✓Auto-discovers all your projects from ~/Desktop and other folders
✓Scores project health — flags stale repos before they become problems
✓Tracks tasks with priority levels and due dates, reminds you of overdue work
✓Stores prompts — save your best AI prompts, sorted by project
✓Runs your console with context already loaded across local models and connected cloud agents
✓Sends a morning digest every weekday at 8am via desktop notification
✓Suggests tasks based on your project health and current workload
✓Enhances prompts with AI before you use them elsewhere

🔒 Privacy first: Only your chat messages and project summaries are sent to Anthropic's API for AI processing. Your actual code, file contents, and database never leave your machine.
~/.devbrain/db.py
The memory — SQLite database storing all your projects, tasks, prompts, chat history and settings.

The watcher — scans project folders, detects tech stacks, counts TODOs, reads git history.

The AI — all Claude API calls live here. Chat, digest generation, project analysis, task suggestions.
~/.devbrain/scheduler.py
The alarm clock — runs background jobs automatically: scans, morning digest, task reminders.

3. Getting started in 3 steps

1
Start Axon

Open a terminal and run:

devbrain

This starts the server and opens your browser automatically at http://localhost:7734

First time? Run source ~/.zshrc first to load the devbrain command into your terminal.

Add your Anthropic API key

Click Settings in the bottom-left sidebar. Paste your API key and click Save settings.

How the health score is calculated
Start at: 100
Git age:
−30 if no commit in 30+ days  ·  −15 if 14+ days  ·  −5 if 7+ days
TODO count:
−20 if 50+ TODOs  ·  −10 if 20+  ·  −5 if 10+
No git repo:
−10 (project not version controlled)

10. CLI commands

devbrain
Start the server and open the browser
devbrain stop
Stop the running server
devbrain restart
Stop then start again (use after settings change)
devbrain status
Check if the server is running and on which PID
devbrain log
Tail the live server log (useful for debugging)
devbrain open
Open the browser without (re)starting the server
🚀 Make Axon start automatically on login

Add this to your ~/.zshrc or ~/.bashrc:

# Auto-start Axon on terminal open
devbrain status >/dev/null 2>&1 || devbrain &

This checks if Axon is running — if not, starts it silently in the background.

11. Connecting to your phone

Axon runs on localhost:7734 — your phone can access it when on the same Wi-Fi network. Here are the three ways to connect your mobile device:

1
Direct Wi-Fi access (easiest)

When your phone and computer are on the same Wi-Fi, your phone can reach Axon directly.

# Find your computer's local IP
ip addr | grep "inet " | grep -v 127

# Example output: inet 192.168.1.45
# On phone: open http://192.168.1.45:7734

Open http://[YOUR-IP]:7734 in your phone's browser. Bookmark it for quick access.

Your IP may change — assign a static local IP in your router settings for reliability.
2
Tailscale — access from anywhere (recommended)

Tailscale creates a private VPN between your devices. Install it on your laptop and phone — then use the Tailscale IP to reach Axon from anywhere in the world, even on mobile data.

💻
Install on laptop
tailscale.com → download for Linux
↔
📱
Install on phone
Tailscale app on Play Store / App Store
# On laptop:
sudo tailscale up
tailscale ip # shows your Tailscale IP e.g. 100.x.x.x

# On phone browser:
http://100.x.x.x:7734

Tailscale is free for personal use (up to 3 devices on free tier, 100 devices on Plus).

3
Add to home screen (PWA-style)

Once you've accessed Axon on your phone via either method above, add it to your home screen for instant access — just like an app.

On Android (Chrome)
Menu → Add to Home screen → Add
On iPhone (Safari)
Share button → Add to Home Screen → Add

12. Connecting OpenAI GPTs / Gemini Gems / Cloud Agents

Axon is your source of truth. You can feed Axon data into other AI tools to supercharge them with your real workspace context. Here's how to connect each platform:

🤖
ChatGPT Projects
Use your Axon context as a project system prompt
①Go to Axon → Console. Ask: "Generate a system prompt summarising all my workspaces and current state for use in ChatGPT"
②Copy the generated text.
③In ChatGPT → create or open a Project → paste it as the Project Instructions.
④Refresh weekly — re-run step ① and update the project instructions with fresh data.
💎
Google Gemini Gems
Create a personalised Gem with your dev context
①Ask Axon Console: "Write a Gemini Gem system prompt for a developer assistant that knows my workspaces"
②Go to gemini.google.com → Gems → Create a Gem
③Paste the prompt. Give it a name like "DevAssist - [Your Name]"
④Now your Gem knows your project landscape. Use it for quick mobile queries.
🟠
Claude Projects (claude.ai)
Feed Axon data into a Claude Project
①Export a workspace snapshot: ask Axon to "summarise all my active workspaces with health scores and open missions in markdown"
②Go to claude.ai → Projects → create "My Dev Context"
③Upload the markdown as a Project Knowledge file
④Now Claude.ai has your context for mobile/browser use — refresh monthly
🔮 Coming soon: A /api/export endpoint that produces a formatted markdown snapshot of all your workspaces and missions — ready to paste into any AI system prompt. Also planned: a webhook endpoint so external tools can push mission updates directly to Axon.

14. Troubleshooting

❌ "devbrain" command not found

The ~/bin directory isn't in your PATH yet. Run:

source ~/.zshrc

Then try devbrain again. If still not found, check that ~/bin/devbrain file exists.

❌ Server starts but AI features say "API key not set"

Go to Settings → paste your Anthropic API key → click Save. The key is stored in the SQLite database, not in any config file.

Get a key at console.anthropic.com

❌ No projects showing up

Check Settings → Workspaces Root. Make sure the path is correct (e.g. ~/Desktop). Then click Axon Scan in the sidebar.

A project is detected if the folder contains any of: package.json, .git, pyproject.toml, requirements.txt, Cargo.toml, go.mod, next.config.js

Bare folders with no code markers won't be picked up — that's by design.

❌ Morning digest not running

The brief scheduler runs inside the Axon server process. If the server isn't running at 8am, the brief won't fire.

Solution: Add to ~/.zshrc:

devbrain status >/dev/null 2>&1 || devbrain &

This auto-starts Axon whenever you open a terminal.

❌ How do I reset everything and start fresh?
devbrain stop
rm ~/.devbrain/devbrain.db
devbrain

This deletes the database (all workspaces, missions, playbooks, history) but keeps all your code and scripts. Axon re-initialises with a clean DB on next start.

❌ Port 7734 is already in use
kill $(lsof -ti:7734) 2>/dev/null
devbrain

This kills whatever is on port 7734 and starts Axon fresh.
